# 温湿度モニタリングシステム iOSアプリケーション (G4)

本リポジトリは、東京国際工科専門職大学の講義「devpro3」における第4班（G4）の成果物です。
Flaskサーバーから不規則な周期で送信される温湿度生データを取得し、iOS端末上でインサイト表示、時系列グラフ表示、カレンダーフィルタリングによる履歴表示を行うクライアントアプリです。

---

## 🛠️ 基本設計 (System Architecture)

大量かつ不規則なデータ流入（数秒〜数時間間隔）に対しても、120fpsでヌルヌル動く「Apple純正アプリ（ヘルスケアなど）」と同等の操作感と堅牢性を両立させるアーキテクチャを採用しています。

### アーキテクチャパターン
* **MVVM (Model-View-ViewModel)** を採用し、UI描画層とデータ処理層を徹底的に分離。

### テクニカルハイライト
1. **QRコードスキャン自動接続 ＆ FaceID連携**
   - `AVFoundation` を利用した爆速QRスキャナー。読み取りと同時に自動で接続処理を開始するシームレスな体験。
   - `LocalAuthentication` による生体認証。IP履歴やカメラ起動などの重要機能へのセキュアなアクセス。
2. **Swift Chartsの極限軽量化（レンダリング最適化）**
   - 1万件を超えるデータでもストールしないよう、点（Point）描画から「滑らかな1本の線（LineMark）＋グラデーション（AreaMark）」へと一括描画ロジックを最適化。
   - 長押し追従には、Appleのコアアルゴリズムである **二分探索（Binary Search）** を導入し、計算量を $O(N)$ から $O(\log N)$ へ劇的に削減。
3. **バックグラウンド分散処理 (Task.detached)**
   - データの「期間別丸め（グループ化）アルゴリズム」をUIスレッドから完全に切り離し、バックグラウンドの独立スレッドで非同期並列計算。

## 📱 UI/UX・デザイン設計 (UI Design)

Appleの **HIG（Human Interface Guidelines）** に準拠し、無駄な装飾を削ぎ落とした、静かでクリーンな純正ライクのUIを追求しています。

| 画面名 | 画面の特徴・こだわり |
| :--- | :--- |
| **サーバー設定画面** | テンキー展開時にキーボードの角に不自然な白隙間が発生しないよう、背景グレー色を全画面のセーフエリア外まで100%敷き詰めるバグ抑止設計。 |
| **概要（ダッシュボード）** | Appleヘルスケアスタイルのマテリアルカード。温湿度のしきい値に基づき、「熱中症防止」「乾燥対策」などのインテリジェントな環境インサイトを動的生成。 |
| **グラフ画面** | 控えめな点線で全体平均（RuleMark）を配置。グラフをなぞると全体平均からのプラスマイナス差分（例: `+1.2℃`）が指に吸い付くように連動。 |
| **履歴画面** | 秒単位データ流入時のUIクラッシュを防ぐため、1日限定のオンデマンド抽出型リストを採用。カレンダーのポップオーバーサイズを320pxに固定し、iOS標準の表示潰れバグを回避。 |
